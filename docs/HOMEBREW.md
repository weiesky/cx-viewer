# Homebrew installation and release guide

CX-Viewer uses this repository as a Homebrew tap. The formula is stored at
`Formula/cx-viewer.rb` and installs the prebuilt package published to npm.

## Install CX-Viewer

Users run:

```bash
brew tap weiesky/cx-viewer https://github.com/weiesky/cx-viewer.git
brew install weiesky/cx-viewer/cx-viewer
```

CX-Viewer requires the OpenAI Codex CLI. On macOS, install it separately when
needed:

```bash
brew install --cask codex
```

Common maintenance commands:

```bash
brew upgrade weiesky/cx-viewer/cx-viewer
brew uninstall cx-viewer
brew untap weiesky/cx-viewer
```

## Maintainer checklist for each release

The Homebrew formula must reference an npm version that already exists. Do not
point it at an unpublished `package.json` version. At the time this guide was
added, npm had `cx-viewer@1.0.4`, while the local source version was `1.0.6`.

### 1. Publish the npm release

1. Make sure `package.json` contains the intended version.
2. Commit the release and create a matching tag such as `v1.0.7`.
3. Push the commit and tag.
4. Publish a GitHub Release for that tag. The existing
   `.github/workflows/npm-publish.yml` workflow publishes the npm package.
5. Wait for the workflow to finish, then confirm the registry version:

```bash
npm view cx-viewer version
```

### 2. Calculate the npm tarball SHA-256

Replace `1.0.7` below with the version that was just published:

```bash
rm -f /tmp/cx-viewer-1.0.7.tgz
npm pack cx-viewer@1.0.7 --pack-destination /tmp
shasum -a 256 /tmp/cx-viewer-1.0.7.tgz
```

Using the npm tarball is intentional: it contains the production `dist/`
assets generated by `prepublishOnly`. A GitHub source archive may not contain
those ignored build artifacts.

### 3. Update the formula

Edit `Formula/cx-viewer.rb`:

```ruby
url "https://registry.npmjs.org/cx-viewer/-/cx-viewer-1.0.7.tgz"
sha256 "SHA256_FROM_THE_PREVIOUS_STEP"
```

The version is inferred from the URL, so a separate `version` field is not
normally needed.

### 4. Validate locally

First run the repository checks that do not require an installed Tap:

```bash
brew style Formula/cx-viewer.rb
```

Homebrew 5 requires Formulae to live in a Tap and no longer accepts a Formula
file path for `brew install` or `brew audit`. After pushing the Formula update,
refresh the Tap and run the full checks by name:

```bash
brew tap weiesky/cx-viewer https://github.com/weiesky/cx-viewer.git
brew update
brew audit --strict weiesky/cx-viewer/cx-viewer
brew reinstall --build-from-source weiesky/cx-viewer/cx-viewer
brew test weiesky/cx-viewer/cx-viewer
```

If an older development copy is already installed by Homebrew, uninstall it
before repeating the installation test:

```bash
brew uninstall cx-viewer
```

The formula compiles the native `node-pty` dependency, so Xcode Command Line
Tools are required on macOS.

### 5. Publish the formula update

Commit and push the formula and documentation changes to the default branch:

```bash
git add Formula/cx-viewer.rb README.md docs/HOMEBREW.md
git commit -m "chore: update Homebrew formula to 1.0.7"
git push origin main
```

Then run the Tap-based audit, reinstall, and test commands from step 4. If they
pass, the Homebrew release is complete.

Existing users then receive the update through:

```bash
brew update
brew upgrade weiesky/cx-viewer/cx-viewer
```

## Optional: create a conventional one-command tap

The current repository is not named with Homebrew's `homebrew-` prefix, so the
first installation uses an explicit `brew tap ... <URL>` command. To support a
single conventional command such as:

```bash
brew install weiesky/tap/cx-viewer
```

manually create a public GitHub repository named `weiesky/homebrew-tap`, copy
`Formula/cx-viewer.rb` into its `Formula/` directory, and keep that copy updated
after each npm release. This separate repository is optional; the installation
method documented above works directly from the existing cx-viewer repository.
