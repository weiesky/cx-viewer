# 辅助请求隔离

## 背景

部分请求详情会展示 `authorization`、`x-api-key` 等敏感认证头。CX Viewer 会在这些 header 旁显示这篇说明，是为了提醒：辅助功能不能随意复用正在运行的 Codex 会话凭据。

## 原则

内部辅助请求应当与 Codex 主对话隔离：

1. 如果辅助功能必须调用模型服务，应使用独立 endpoint 和独立凭据。
2. 不要把主会话的 `authorization` header 转发给无关服务。
3. prompt、文件片段和工具上下文都应视为用户数据，启用代理 profile 后可能经过对应服务。
4. UI 级功能能本地处理时优先本地处理。

## 为什么重要

复用会话凭据可能让无关辅助流量绑定到用户的主 agent 会话，也可能把请求上下文泄露给非预期服务。将辅助调用隔离后，代理行为更容易审计，也能避免 UI 功能与 Codex turn 之间发生意外串扰。
