# Teammate

## Визначення

Teammate — це колаборативний agent у режимі команди Claude Code Agent Team. Коли головний agent створює команду через `TeamCreate` і генерує teammate за допомогою інструменту `Agent`, кожен teammate працює як незалежний процес agent із власним контекстним вікном та набором інструментів, спілкуючись з членами команди через `SendMessage`.

## Відмінності від SubAgent

| Характеристика | Teammate | SubAgent |
|------|----------|----------|
| Життєвий цикл | Існує постійно, може отримувати кілька повідомлень | Одноразове завдання, знищується після завершення |
| Спосіб комунікації | Двонаправлені повідомлення через SendMessage | Односпрямований виклик батько→дитина, повернення результату |
| Контекст | Незалежний повний контекст, зберігається між раундами | Ізольований контекст завдання |
| Модель співпраці | Командна співпраця, можуть спілкуватися між собою | Ієрархічна структура, взаємодія лише з батьківським agent |
| Тип завдань | Складні багатокрокові завдання | Одиничні завдання: пошук, дослідження тощо |

## Поведінка

- Створюється головним agent (team lead) через інструмент `Agent` з призначенням `team_name`
- Спільний список завдань через `TaskList` / `TaskGet` / `TaskUpdate`
- Після кожного раунду виконання переходить у стан idle, очікуючи нове повідомлення для активації
- Може бути коректно завершений через `shutdown_request`

## Опис панелі статистики

Панель статистики Teammate відображає кількість API-викликів кожного teammate. Стовпець `Name` містить ім'я teammate (наприклад, `reviewer-security`, `reviewer-pipeline`), стовпець `Кількість` — загальну кількість API-запитів, згенерованих цим teammate.
